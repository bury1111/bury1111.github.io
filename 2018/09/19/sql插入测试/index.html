sql插入测试

发表于 2018-09-19 |

功能 ：测试sql server、oracle、mysql不同数据量下，1-300000条不同数据量的入库效率。

TSL.exe： 64位 DOS 执行 时间单位：秒(S) 时间记录方式：mtic\mtoc

场景：

  1. 1、数据库：sql server、oracle、mysql
  2. 2、模式：API、ADO
  3. 3、入库数据级别：1-300000
  4. 4、特殊字段：二进制
  5. 5、插入方式：sql串拼接、tsl数组分批插入

批量拼接sql整串（ 最大允许字符串长度** SQL server：8000 个字符； Mysql 默认 1M；Oracle：6000 **个字符 ）

  • 拼接方式：insert into table(field1,field2) values(v1,v2);insert into table(field1,field2) values(v1,v2);……

批量拼接sql value值(最大允许字符串长度: SQL server：8000个字符；Mysql默认1M；Oracle：不支持这种写法；最大允许values的项数：1000)

  • 拼接方式：insert into table(field1,fied2) values(v1,v2),(v1,v2),(v1,v2)……

测试使用插入语句

insert into test_sqlserver(id,name,c1,c2,c3,c4,c5,c6,c8,c9) values(1,'"$random()$"xx业务界面','12','342','2048','2018-08-21 14:51:20.000','2018-08-28 14:00:00.000','bury','测试插入','测试插入');

得出结论：三种数据库，sql server的插入效率要高一些；

  1. 无索引，无二进制数据的情况下推荐使用 API模式下的Execsql方式中批量拼接sql value的方法进行批量插入；
  2. 无索引，有二进制数据的情况下推荐使用API模式下的TSL方式进行批量插入；
  3. 有索引，无二进制数据的情况下推荐使用API模式下的Execsql方式中批量拼接sql value的方法进行批量插入；
  4. 有索引，有二进制数据的情况下推荐使用API模式下的TSL方式进行批量插入；
